Generated by repo scan on 2026-06-01. Last updated: 2026-06-02.

## Project purpose

This repository builds materials-science knowledge graphs from research papers. Main workflow:

1. Download or collect PDFs in `polymer_papers/`.
2. Extract schema-aligned terminology from PDFs with an LLM in `app/modules/extract_terms.py`.
3. Convert extracted terms JSON into MatKG graph JSON with `app/modules/json2kg.py`.
4. Query graph plus PDF evidence through KG-RAG in `app/modules/kg_rag_api.py`.
5. Evaluate generated graph files with scripts in `scripts/`.

## CBORG API call locations

CBORG is called through the OpenAI-compatible Python SDK, not by raw `requests.post`.

### Primary implementation: `app/modules/extract_terms.py`

Call chain:

1. `run_extraction(..., backend="cborg", cborg_base=..., cborg_api_key=...)`
2. `make_chat_client(...)`
3. `CBorgChatClient(...)`
4. `CBorgChatClient.chat(...)`
5. `self.client.chat.completions.create(...)`

Relevant code:

```python
class CBorgChatClient(ChatClient):
    def __init__(self, model: str, api_key: Optional[str] = None, base_url: Optional[str] = None):
        self.model = model
        self.client = openai.OpenAI(
            api_key=api_key or os.environ.get("CBORG_API_KEY"),
            base_url=(base_url or os.environ.get("CBORG_BASE_URL") or "https://api.cborg.lbl.gov").rstrip("/"),
        )

    def chat(self, prompt: str, *, temperature: float = 0.0, timeout: int = 240) -> str:
        resp = self.client.chat.completions.create(
            model=self.model,
            messages=[{"role": "user", "content": prompt}],
            temperature=temperature,
            timeout=timeout,
        )
        return resp.choices[-1].message.content or ""
```

Backend selection:

```python
if b in ("cborg", "cborg-openai"):
    return CBorgChatClient(model=model, api_key=cborg_api_key, base_url=cborg_base)
```

## Environment notes

`PYSTOW_HOME` is set in `.env` as `PYSTOW_HOME=.cache/pystow` so PyStow uses a repo-local writable cache directory without hardcoding a user path. `linkml-runtime` is installed, but importing `linkml_runtime` can fail if PyStow tries to create/use a default home directory that is not writable by the current user. Scripts that call `load_dotenv()` (`app/modules/extract_terms.py`, `app/run_pipeline_cborg.py`, `scripts/test_chat_apis.py`) can pick up this setting.

`KG_RAG_RETRIEVAL_BACKEND=lexical` is set in `.env` for CLI/OpenWebUI chat. This avoids loading FAISS/Torch/SentenceTransformer on Python 3.14, where native ML stack startup/shutdown can segfault or leak loky semaphores. Semantic retrieval remains available with `KG_RAG_RETRIEVAL_BACKEND=semantic` when the embedding model and native dependencies are stable.

KG-RAG chat also uses `KG_RAG_LLM_TIMEOUT=120` and `KG_RAG_SHOW_BASELINE=0` by default. CLI chat now makes one KG-RAG LLM call per question unless `--show-baseline` is passed. Missing local PDFs referenced by a larger KG are skipped at debug level instead of warning, so chat output is not flooded when `storage/kg/matkg_qwen3_235b_580papers.json` points to PDFs absent from local `polymer_papers/`.
